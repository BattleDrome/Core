Welcome to BattleDrome Alpha!

The long-awaited alpha is here! It was a long road, but we're super-excited to have the alpha out in the wild and able to be tested by the public!

Please keep in mind, that the alpha WILL HAVE BUGS! It's an Alpha, it's meant to be tested in the wild so we can find the issues/bugs.

Here is a list of a few things to be aware of going into the test (for a full list of known issues, please see the issues section in the github project):

  • The UI is not 100% complete, it's a rough UI for now which just gives the basic functionality. It will be significantly improved in future versions.
  • The UI does NOT include the graphical spectator engine, this was not planned for this phase, and is planned for a later release
  • The UI is a bit sluggish. We are aware it's a bit clunky, right now this has to do with some limitations in the underlying frameworks. We are expecting a future patch to Web3 and Truffle to allow us a much more elegant way of fetching/syncing/updating UI elements from changes in the blockchain.
  • Gas prices are high. We know this, we've optimized them heavily in many areas. The big one is in the spectator/polling of events. This is compensated in the polling reward, but we do want to reduce the cost in order to improve the overal economic efficiency of the game. This will come in future versions (in some cases due to underlying technology improvements, in other cases due to further optimization passes we undertake)
  • Currently the dialing/balance of the combat/economics are not fully tested. This is a "first stab". We expect to tune these parameters throughout the Alpha and Beta in order to achieve a much more balanced set of mechanics for the final release.
  • YES! We are aware the gender of a warrior is set randomly at creation time, and you have no control. This is intentional... What is not intentional is that the name can't be set after creation, (or rather we hadn't realised this was an issue until later in alpha development, we WILL fix this, but it's low priority for now, so please use gender neutral names if it bugs you!)
  • We are aware that the graphical rendering of the warriors has some minor glitches. Again, this was meant to be a basic html interface, without any graphics in the original scope, we wanted to add a bit more flair, but the fine-tuning of rendering was low-priority for alpha. We will be refining leading up to beta.

Keeping these things in mind, please get into the game, try it out, and have fun! And don't hesitate to send us your feedback via github!

Since the Alpha code is now open-source, please don't hesitate to give us PRs for improvements if you feel motivated to participate. We continue to develop in our private development environment, but will be incorporating PRs and bugfixes directly in github. Then when the next major release is merged into the gitlab environment, we'll back-port any valuable additions/improvements from the community!

Instructions

Please follow these instructions to participate in the alpha, and ensure you have a (hopefully) smooth experience:

  • Install Metamask in your browser and set it up according their setup instructions
  • DO NOT use your live ethereum main-net account for interacting with BattleDrome
  • Ensure you're targeting the right network. Keep in mind BattleDrome currently runs on the Rinkeby Test Net
  • Get some Ether into your account, on Rinkeby this can be done via the Rinkeby Faucet at: https://faucet.rinkeby.io/ you need to take the address for an account generated by metamask, and share it via Twitter or some other accepted social network, then use the web form to request ether from the faucet.
  • Now access this page, ensure your metamask den is unlocked (the page should prompt you with a notice if it is not)
  • At this point you can use the tabs at the top of the page to navigate objects such as Warriors and Events, and use the Create buttons to begin creating your first Warrior or Event!

Further instructions will be created during the alpha.
